02. css 선언 방식

  • 내장 방식: html <head> 요소에 작성 / 내용 (Content) 로 스타일을 작성하는 방식
    • html 파일 내에서만 작성하면 css 파일 유지보수하기 어려움 (권장 x)

  • 링크 방식: <link/> 로 외부 css 문서 main.css 를 가져와서 연결하는 방식 (= 병렬 방식)

  • 인라인 방식: 요소의 style 속성에 직접 스타일을 작성하는 방식 (선택자 x)

    • 매 수정시 덮어쓸 상황에 유지보수 어려움 / 우선순위 개념에 있어서도 권장 x

  • @import 방식: css 의 임포트 규칙으로 css 문서 안에서 또 다른 css 문서를 가져와 연결하는 방식

    • 연결순서: html ← main.css ← box.css
      • main.css 가 연결이 먼저되어서 html 에 해석이 끝나야지만 box.css 가 연결되는 구조!
        • 단점: 직렬 연결 방식은 실제로 지연될 수 있음 (일반적으로 많이 사용되지 x)

03. css 선택자_기본

  1. 전체 선택자 (Universal Selector) * {} - 모든 요소를 선택!
  1. tag 선택자 (Type Selector) ABC {}
    • tag 이름이 ABC <li>인 요소 선택!
  1. class 선택자 .ABC {}
    • HTML class name (속성)의 값이 ABC .orange 인 요소 선택!
  1. ID 선택자 #ABC {}
    • HTML id 이름값이 ABC #orange 인 요소 선택!

04. css 선택자_복합

  1. 일치 선택자 (Basic Combinator) ABCXYZ tagName.className

    • 선택자 ABC span 와 XYZ orange 를 동시에 만족하는 요소 선택!

  2. 자식 선택자 (Child Combinator) ABC > XYZ

    • 선택자 ABC <ul> 의 자식 요소 XYZ .orange 선택! —> 좀 더 특정한 선택이 필요할 경우

  3. 하위 (후손) 선택자 (Descendant Combinator) ABC XYZ

    • 선택자 ABC 의 하위요소 xyz 선택! —> 일반적으로 편하게 사용

  4. 인접 형제 선택자 (Adjacent Sibling Combinator) ABC + XYZ (중요!)

    • 선택자 abc 의 다음 형제 요소 xyz 하나를 선택!

  5. 일반 형제 선택자 (General Sibling Combinator) ABC ~ XYZ

    • 선택자 abc 의 다음 형제 요소 xyz 모두를 선택!

05. css 가상 클래스 선택자 (Pseudo Classes)

  1. HOVER ABC:hover

    • 선택자 요소에 마우스 커서가 올라가 있는 동안 선택!

  2. ACTIVE ABC:active

    • 선택자 요소에 마우스를 클릭하고 있는 동안 선택!

  1. FIRST CHILD ABC:first-child

    • 선택자 abc 가 형제 요소 중 첫째라면 선택!

    .fruits span:first-child {color:red;}

  2. LAST CHILD

    • 선택자 abc 의 형제 요소 중 막내라면 선택

  3. NTH CHILD

    .fruits *:nth-child(2) {color: red;}

    *: —> select 전체

  4. 부정 선택자 ABC:not(XYZ)

    • 선택자 xyz 가 아닌 abc 요소 선택

    .fruits *:not(span) {color: red;}

.*className:*hover {}

06. css 가상 요소 선택자 (Pseudo Elements)

  1. ABC::before

→ 선택자 abc 요소의 내부 앞에 내용 (Content) 을 삽입.

  1. ABC::after

따라서 inline 요소에서 block 요소로 바꿔줘야 한다.

전환할때 쓰는 속성: display: block;

01. block, inline

block 요소들은 가로 사이즈가 따로 정해지지 않아서 auto 라는 기본값이 사용된다면,

부모 요소의 크기만큼 최대한 늘려서 사용하려고 시도를 한다.

그에 다르게, 세로 너비같은 경우엔,

inline 요소와 똑같이 최대한 줄여서 사용하려고 하는 특징을 가지고 있다.

block 요소와 inline 요소의 큰 특징:

inline 요소는 가로 너비가 최대한 줄어들려고 하고, (<span></span>)

block 요소는 가로 너비가 최대한 늘어나려고 한다. (<div></div> )

02. max-width, max-height

요소가 커질 수 있는 최대 가로/세로 너비

  • none 최대 너비 제한 없음 (기본값)
  • auto 브라우저가 너비를 계산
  • 단위 px, em, vw 등 단위로 지정

03. min-width, min-height

요소가 작아질 수 있는 최소 가로/세로 너비

  • 0 최소 너비 제한 없음 (기본값)
  • auto 브라우저가 너비를 계산
  • 단위 px, em, vw 등 단위로 지정

width, height 속성의 기본값은? auto

09. position 배치

→ 요소의 위치 지정 기준

static 기준 없음

relative 요소 자신을 기준 으로 배치!

absolute 위치 상 부모 요소를 기준으로 배치! (위치 상 부모 요소를 꼭 확인해야 함!) **가장 많이 사용하는것!

fixed 뷰포트 (브라우저)를 기준

html 구조 상의 부모 요소를 (화면에 실제 그려지는) 위치 상의 부모 요소로 만드는 가장 무난한 포지션의 값은 relative 를 사용하는 것이다.

따라서 position: absolute; 라는 부모 요소를 기준으로 배치되는 값을 사용했을때는,

그 부모 요소라는 개념이 html 구조 상의 부모 요소가 아닌 (실제 그려지는) 위치 상의 부모 요소라는 것을 잘 기억해두기!

위치 상의 부모란?

실제 화면에 그려지는 곳.

→ ex. .container {} 라는 부분을 확인해서 이 부분이 위치 상의 부모 요소가 될 수 있는지 먼저 code 를 확인을 해야한다.

그 안에 position: ; 값이 없기 때문에 ‘위치 상의 부모 요소로는 적합하지 않다’ 라고 판단한다.

그 다음, 그 상위 요소인 .wrap {} 로 한 뎁스 올라가서 position: ; 값이 있는지 확인한다.

position: ; 값이 있기 때문에 .wrap {} 이 적합하다고 판단하는 것이다.

10. 요소 쌓임 순서 Stack Order

→ 어떤 요소가 사용자와 더 가깝게 있는지 (위에 쌓이는지) 결정

아래 3가지 조건에 부합해야 한다!

조건 1/ 요소에 position 속성의 값이 있는 경우, 더 위에 쌓인다. (기본값 static 제외)

조건 2/ 1번 조건이 같은 경우, z-index 속성의 숫자 값이 높을 수록 위에 쌓인다.

조건 3/ 1번과 2번 조건까지 같은 경우, html 의 구조가 더 나중에 작성되어 있을 수록 위에 쌓인다.

아래 코드에 있는 item 1,2,3 을 비교했을때,

조건 1을 보면 position 값이 셋 다 있기 때문에 다음 조건 2로 넘어가보자.

조건 2에서는 z-index 값이 item 2 는 없기 때문에 제외시키고, item 1 과 item 3 둘 다 값이 동일하기 때문에 그 다음 조건 3으로 넘어가보자.

조건 3에서는 html 구조가 더 나중에 쓰여진 값이 가장 위에 쌓이기 때문에 최종적으로 item 3 이 가장 위에 쌓여 있는 것이다.
